Introdução a programação em C

História:

A linguagem C foi criada em 1970 por Dennis Ritchie, no Bell Laboratories, como uma evolução da linguagem B. Ela tem suas raízes em duas linguagens anteriores: Algol 68 e BCPL.
Seu grande marco aconteceu quando Dennis Ritchie e Ken Thompson reescreveram o código do Unix, utilizando C, o que levou à criação do sistema operacional que conhecemos hoje como Linux.
Nos anos 80, C passou a ser amplamente reconhecida como uma linguagem de propósito geral, com diversos compiladores desenvolvidos por diferentes fabricantes, e até hoje C continua sendo essencial em áreas como sistemas operacionais e firmware.

Importância:

A linguagem C é muito importante pois é eficiente, permitindo controlar o hardware do computador. Ela é uma linguagem simples, e uma base para as outras, se tornando assim fundamental.
Ela também permite a programação de microcontroladores, que são sistemas eletrônicos compactos semelhantes a computadores. Sua manipulação direta da memória otimiza o desempenho, permite reutilização de códigos e oferece portabilidade entre diferentes plataformas.

Estrutura de um Programa em C

Bibliotecas:

Para estruturarmos um programa em C, primeiro precisamos adicionar suas bibliotecas. As bibliotecas possibilitam a criação do código, pois são um conjunto de funções e recursos que facilitam o desenvolvimento do software. As mais usadas são: stdlib. h., stdio. h. (funções de entrada e saída).

Função main e return 0:

Ela é o que inicia e controla o programa. Dentro dela adicionamos todo o código, como variáveis, desenvolvimento e saída de dados.

Exemplo:

 

#include stdio.h
    int main() {
    // codigo do programa
    return 0;
    //Indica que o programa foi  finalizado com sucesso
}

Como compilar e executar um programa em C

Para executar um código em C, precisamos usar um compilador. Para isso temos alguns compiladores gratuitos, como:

  • DEV C++

Sintaxe

Com a sintaxe, aprendemos as regras e estruturas para criar um programa funcional e organizado. Um princípio básico que devemos aprender são as boas práticas.

  • Fazer códigos comentados, para isso utilizamos “// comentário”.
  • Codar de forma estruturada.
  • Organize seu código, pois será mais fácil corrigir possíveis erros.
  • Utilize nomes simples, ou palavras chave para a criação de variáveis
    •

Exemplo de código com boas práticas:

 

//Inclusão de Bibliotecas
#include stdio.h
#include stdlib.h
                    
//Função Main
int main ()
{
    //Mensagem que irá ser mostrada na tela
    printf ("Olá Mundo");
    //Finalização do Programa
    return 0;
}
